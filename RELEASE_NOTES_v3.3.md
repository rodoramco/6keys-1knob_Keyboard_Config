# v3.3.0 — Mouse-aware macros

## Highlights

- **Step-list macro builder** replaces the old single-textarea Macro editor. Compose a macro from an ordered sequence of steps:
  - **Type Text** — literal text (no shortcut interpretation)
  - **Key Combo** — Ctrl/Shift/Alt/Win + key
  - **Mouse Click** — Left / Right / Middle / Double Left, at cursor or fixed X,Y screen coords
  - **Mouse Wheel** — Up / Down with tick count, or Wheel Click
  - **Wait** — pause N ms before the next step
- **AutoHotkey v2 helper** generated on demand from the new **🖱 Macro Helper (AHK)** sidebar card. Run the `.ahk` file alongside the keypad and your mouse macros work end-to-end.
- **Per-slot trigger combos** — macro-typed slots are programmed to emit a hidden Ctrl+Alt+Shift+F13–F21 combo that the AHK helper intercepts. Other slot types (Shortcut / Media / App) are unchanged.
- **Backwards-compatible profile loading** — v3.2 profiles with the old `macro: "..."` string field still load and display normally; on first save in v3.3 they migrate to the new `steps` array (the legacy field is dropped automatically).

## Why is mouse playback handled by AHK?

The MINI KeyBoard dongle's firmware understands only HID Keyboard (`type=0x01`) and HID Consumer Control (`type=0x02`). It has no Mouse usage page in its descriptor and no `type` byte for mouse events. Rather than fight the firmware, v3.3 makes the configurator emit a hidden trigger combo for each macro slot and lets a small AutoHotkey v2 script handle the rich playback (clicks, wheel, mixed sequences). One script, regenerated whenever you change a macro.

## What's new under the hood

- New `steps[]` schema on each Macro assignment.
- `encodeAssignment(d, kid)` now accepts the slot id so it can emit the per-slot trigger combo for `type === 'macro'`.
- Added `F13..F24` to the HID keycode map (codes `0x68..0x73`).
- New `.ahk` exporter with proper AHK v2 syntax (`SendText`, `Click`, `Sleep`, modifier prefixes, escape rules for backticks/quotes/newlines).
- New CSS for the step-list builder UI (drag-free reordering with ▲/✕/▼ controls, type-coloured pill badges, focus-preserving inputs).

## Breaking changes

None. Old profiles open and run; the only visible change is that the Macro tab now shows a step list instead of a textarea. A textbook `Hello World` macro from v3.2 becomes a single **Type Text** step the first time you open it.

## Known limitations

- Mouse playback requires AutoHotkey v2 running on the host. Without it, macro-typed pad keys produce a hidden combo with no visible effect.
- Trigger combos use Ctrl+Alt+Shift+F13–F21 — extremely unlikely to collide with real-world hotkeys, but if you have something on those exact combos, change the assignments before pressing the macro key.
- Coords for fixed-XY clicks are screen-absolute (top-left origin). If you change resolutions or move the target window, regenerate the macro.

## Installation / upgrade

1. Download `keypad-configurator-v3.3.html` from this release.
2. Open it in Chrome or Edge (WebHID required for sending to the device).
3. Existing profiles will load from `localStorage` automatically. Old `macro` strings auto-migrate to the new step model on first save.
4. For mouse macros, install [AutoHotkey v2](https://www.autohotkey.com/) and run the `.ahk` file generated by the **Macro Helper (AHK)** card.

The previous v3.2.2 build is still available as the **v3.2.2** release on this repo for users who don't need mouse macros.
